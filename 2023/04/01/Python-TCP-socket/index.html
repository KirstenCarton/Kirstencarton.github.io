threading模块用法

python3有一个老旧的_thread模块，不太行，所以直接改用更强大的threading模块

threading 模块除了包含 _thread 模块中的所有方法外，还提供的其他方法：

  • threading.currentThread(): 返回当前的线程变量。
  • threading.enumerate(): 返回一个包含正在运行的线程的list。正在运行指线程启动后、结束前，不包括启动前和终止后的线程。
  • threading.activeCount(): 返回正在运行的线程数量，与len(threading.enumerate())有相同的结果。

除了使用方法外，线程模块同样提供了Thread类来处理线程，Thread类提供了以下方法:

  • run(): 用以表示线程活动的方法。

  • start():启动线程活动。

  • join([time]): 等待至线程中止。这阻塞调用线程直至线程的join() 方法被调用中止-正常退出或者抛出未处理的异常-或者是可选的超时发生。

  • isAlive(): 返回线程是否活动的。

  • getName(): 返回线程名。

  • setName(): 设置线程名。

一个布尔值，表示这个线程是否是一个守护线程（True）或不是（False）。 这个值必须在调用 start() 之前设置，否则会引发 RuntimeError 。它的初始值继承自创建线程；主线程不是一个守护线程，因此所有在主线程中创建的线程默认为 daemon = False。当没有存活的非守护线程时，整个Python程序才会退出。

守护进程

定义

守护进程是生存期长的一种进程。它们独立于控制终端并且周期性的执行某种任务或等待处理某些发生的事件。他们常常在系统引导装入时启动，在系统关闭时终止。

特性

  • 在后台运行。
  • 与其运行前的环境隔离开来。这些环境包括未关闭的文件描述符、控制终端、会话和进程组、工作目录以及文件创建掩码等。这些环境通常是守护进程从执行它的父进程（特别是shell）中继承下来的。
  • 启动方式特殊，它可以在系统启动时从启动脚本 /etc/rc.d 中启动，可以由 inetd 守护进程启动，可以由 crond 启动，还可以由用户终端（通常是 shell）执行。

总之，除开这些特殊性以外，守护进程与普通进程基本上没有什么区别。因此，编写守护进程实际上是把一个普通进程按照上述的守护进程的特性改造成为守护进程。

socket模块用法

  • 什么是 Socket?

Socket又称”套接字”，应用程序通常通过”套接字”向网络发出请求或者应答网络请求，使主机间或者一台计算机上的进程间可以通讯。

Socket（套接字）是一种网络编程中的抽象概念，它提供了一种通信机制，允许两个或多个进程在网络中传递数据。在计算机网络中，Socket通常指一个IP地址和一个端口号的组合。

在网络编程中，Socket套接字通常被用来建立客户端和服务器之间的通信。客户端和服务器通过Socket进行通信，客户端创建一个Socket并连接到服务器，服务器则通过接收和处理客户端的请求来响应客户端。Socket提供了一个抽象层，使得网络编程可以隐藏底层网络协议的细节。

Socket套接字的工作方式基于两种协议：传输层协议和网络层协议。传输层协议包括TCP（传输控制协议）和UDP（用户数据报协议），它们负责在端到端之间提供可靠的通信服务。网络层协议负责在不同的网络之间进行数据传输。

在网络编程中，使用Socket套接字需要经历以下步骤：

  1. 创建一个Socket
  2. 绑定Socket到一个IP地址和端口号
  3. 监听连接请求
  4. 接受连接请求
  5. 发送和接收数据
  6. 关闭连接

在创建一个Socket时，需要指定传输层协议、网络层协议、Socket类型和Socket选项。Socket类型可以是面向连接的（如TCP）或无连接的（如UDP）。Socket选项可以包括超时、缓冲区大小、重传等设置。

在建立连接后，数据传输可以通过两种方式进行：阻塞和非阻塞。阻塞方式会在数据传输完成之前一直等待，而非阻塞方式会在数据没有准备好时立即返回。通常，使用非阻塞方式可以提高程序的效率和响应性能。

socket是什么？套接字是什么？ - 知乎 (zhihu.com)

函数 socket.socket 创建一个 socket，该函数带有两个参数：

  • Address Family：可以选择 AF_INET（用于 Internet 进程间通信） 或者 AF_UNIX（用于同一台机器进程间通信）,实际工作中常用AF_INET
  • Type：套接字类型，可以是 SOCK_STREAM（流式套接字，主要用于 TCP 协议）或者 SOCK_DGRAM（数据报套接字，主要用于 UDP 协议）

AF_INET和AF_UNIX都是Socket编程中常用的地址族（Address Family），它们用于指定Socket的地址类型和网络协议族。

AF_INET是用于IPv4地址的套接字地址族，它使用32位的IP地址和16位的端口号来标识网络上的进程，常用于TCP和UDP网络通信。在使用AF_INET时，需要指定Socket的类型（SOCK_STREAM（针对tcp的）或SOCK_DGRAM（针对udp的））、协议（TCP或UDP）以及IP地址和端口号。

AF_UNIX则是用于Unix域套接字的套接字地址族，它是基于文件系统的本地套接字，用于在同一台计算机上的进程之间进行通信。在使用AF_UNIX时，Socket地址是由一个路径名来标识的，而不是IP地址和端口号。

因此，AF_INET和AF_UNIX的主要区别在于：

  1. AF_INET是用于网络通信的地址族，而AF_UNIX是用于本地进程间通信的地址族。

  2. AF_INET使用IP地址和端口号来标识网络上的进程，而AF_UNIX使用文件系统路径名来标识本地进程。

  3. 在使用AF_INET时，需要指定Socket的类型（SOCK_STREAM或SOCK_DGRAM）和协议（TCP或UDP），而在使用AF_UNIX时，Socket的类型和协议是固定的。

总的来说，AF_INET适用于网络通信，而AF_UNIX适用于本地进程间通信，它们都有各自的优缺点和适用场景。
