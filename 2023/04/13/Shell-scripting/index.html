  • find命令用于在指定目录中查找文件和目录。
  • /表示从根目录开始查找。
  • -type f指定只查找文件，而不是目录或其他类型的文件。

  • ls -l {} ;是要执行的命令，其中{}表示查找到的文件名，;表示命令的结尾。ls -l是列出文件的详细信息，包括文件的权限、所有者、大小等等。

这里的;是个转义字符

-exec 表示执行命令

具体来说，命令的各个部分的含义如下：

  • find .: 在当前目录(.)下查找文件和目录。
  • -exec /bin/bash ;: 对查找到的每个文件和目录执行/bin/bash命令，并用;表示命令的结束符号。这里的;用来告诉find命令，在执行完/bin/bash命令后立即退出，而不是继续执行查找任务。
  • -quit: 在执行完第一个/bin/bash命令后退出find命令，不再继续查找其他文件和目录。因此，这个命令实际上是在当前目录下执行了一个/bin/bash命令，并在执行完第一个命令后退出find命令。如果当前目录中有多个文件和目录，find命令会查找到所有的文件和目录，但只会执行第一个文件或目录的/bin/bash命令，而后面的文件和目录会被忽略。

sync

在 Linux 中，sync 是一个命令行工具，用于将文件系统中的所有未写入的缓存数据强制刷新到磁盘上，以确保数据的一致性。这个命令可以用于各种情况下，比如在系统关闭前，强制所有挂载的文件系统写入磁盘中的数据，或在进行磁盘备份之前，确保数据已经被完全写入到磁盘上。

sync 命令不需要任何参数，只需在命令行中输入sync，系统就会将所有未写入的缓存数据写入磁盘中。使用sync 命令可以保证数据在掉电或系统崩溃等情况下不会丢失。

另外，sync 命令也可以与其他命令组合使用，例如：

  • sync && reboot：在重启系统之前强制刷新所有挂载的文件系统中的数据。
  • sync && echo 3 > /proc/sys/vm/drop_caches：刷新缓存中的数据并清空系统内核缓存。

ifconfig

在计算机网络中，广播地址通常是根据网络地址和子网掩码自动分配的，而不是手动配置的。

在 IPv4 协议中，广播地址的计算方法是将网络地址的所有位都设置为 1，然后将主机地址的所有位都设置为 0。例如，在一个 IP 地址为 192.168.1.0，子网掩码为 255.255.255.0 的子网中，广播地址为 192.168.1.255。这个计算过程是根据子网掩码中的 0 和 1 来判断网络地址和主机地址的。

在 IPv6 协议中，广播地址是一个预定义的地址，即 FF02::1，用于向一个子网内的所有节点发送数据包。IPv6 不支持网络地址和主机地址的划分，因此不需要计算广播地址。

虽然可以手动配置广播地址，但是这种情况很少出现，因为手动配置需要知道网络地址和子网掩码的详细信息，而且容易出错。一般来说，计算机网络中的广播地址是根据自动分配的规则计算得出的

eth0是一个Linux系统中的网络接口，用于表示第一个以太网接口，以太网是一种常见的局域网协议，通常使用网线来连接计算机和路由器等设备。

在Linux系统中，每个网络接口都有一个对应的名称，以eth开头，后面跟着一个数字，表示第几个以太网接口。eth0是指第一个以太网接口，而eth1、eth2等依此类推。

ICMP（Internet Control Message Protocol）是互联网协议族中的一个子协议，用于向网络中的其他设备发送错误消息和状态信息，以便于网络的故障诊断和管理。

ICMP消息通常由网络设备（如路由器、交换机）或主机（如计算机）发送，用于诊断网络故障、测试网络连接性、维护网络状态等。

ICMP消息由报文头和数据组成，报文头包含类型（Type）、代码（Code）、校验和（Checksum）等字段。ICMP消息的类型和代码共同决定了消息的目的和内容。

常见的ICMP消息类型和用途如下：

回显请求（Echo Request）和回显回答（Echo Reply）：用于测试网络连接性，常用于ping命令中。

目的不可达（Destination Unreachable）：用于指示网络中的某个设备或主机不可达或已关闭。

超时（Time Exceeded）：用于指示某个数据包已经在网络中传输了太长时间，可能已经丢失或被损坏。

重定向（Redirect）：用于指示网络中的某个设备或主机不应该向某个目标发送数据包，而应该发送到另一个设备或主机。

路由器通告（Router Advertisement）和路由器请求（Router Solicitation）：用于IPv6网络中的路由器自动配置和发现。

ICMP协议通常运行在IP协议的上层，不需要进行端口号的设置。在网络故障诊断和管理中，ICMP是一种非常重要的工具，可以帮助管理员快速定位和解决网络问题。

Linux ps （英文全拼：process status）命令用于显示当前进程的状态，类似于 windows 的任务管理器。

  • -A 列出所有的进程
  • -w 显示加宽可以显示较多的资讯
  • -au 显示较详细的资讯
  • -aux 显示所有包含其他使用者的进程

-ef是

ps命令的选项之一，它用于显示所有进程的详细信息，包括进程的UID、PID、PPID、C、STIME、TTY、TIME、CMD等。

  • UID表示进程的用户标识符，即进程的所有者。

  • PID表示进程的标识符，是唯一的，可以用来识别进程。

  • PPID表示父进程的标识符，即创建该进程的进程。

  • C表示进程使用的CPU资源百分比。

  • STIME表示进程启动的时间。

  • TTY表示进程所在的终端设备。

  • TIME表示进程运行的时间。

  • CMD表示启动该进程的命令。

  • au(x) 输出格式 :

  • USER: 行程拥有者

  • PID: pid

  • %CPU: 占用的 CPU 使用率

  • %MEM: 占用的记忆体使用率

  • VSZ: 占用的虚拟记忆体大小

  • RSS: 占用的记忆体大小

  • TTY: 终端的次要装置号码 (minor device number of tty)

  • STAT: 该行程的状态:

  • D: 无法中断的休眠状态 (通常 IO 的进程)

  • R: 正在执行中

  • S: 静止状态

  • T: 暂停执行

  • Z: 不存在但暂时无法消除

  • W: 没有足够的记忆体分页可分配

  • <: 高优先序的行程

  • N: 低优先序的行程

  • L: 有记忆体分页分配并锁在记忆体内 (实时系统或捱A I/O)

  • START: 行程开始时间

  • TIME: 执行的时间

  • COMMAND:所执行的指令

kill

在kill命令中，-9是一个参数，表示要发送SIGKILL信号给指定的进程。SIGKILL信号是一种无法被捕获或忽略的信号，它能够强制终止进程，即使进程在执行期间没有做任何清理工作。

杀死指定用户所有进程

这个命令的作用是使用ps -ef | grep php命令查找所有名称中包含php的进程，并将结果传递给kill命令，使用-9参数向这些进程发送SIGKILL信号，强制终止它们的执行。

具体而言，命令中的$()语法表示将命令的输出作为参数传递给kill命令。ps -ef用于列出系统中所有进程的详细信息，而grep php用于从中筛选出包含php字符串的进程。

需要注意的是，使用kill -9命令会强制终止进程，并可能会导致数据损坏或丢失。在终止进程之前，最好先尝试使用kill命令发送SIGTERM信号，这样可以让进程有机会完成清理工作并安全退出。

awk

awk基本用法如下：

  1. 打印整个文件：

awk ‘{print}’ filename这个命令会将整个文件的内容打印出来。

  1. 打印指定列：

awk ‘{print $1,$3}’ filename这个命令会打印文件中第一列和第三列的内容，使用空格作为分隔符。

  1. 根据指定条件打印：

awk ‘$1 == “hello” {print $2}’ filename这个命令会打印文件中第一列为“hello”的行的第二列内容。

  1. 使用函数和条件：

awk ‘{if($1 == “hello”) print toupper($2)}’ filename这个命令会打印文件中第一列为“hello”的行的第二列内容，并将其转换为大写。

  1. 计算文件行数：

awk ‘END{print NR}’ filename这个命令会计算文件的行数并打印出来。

  1. 计算文件总和：

awk ‘{sum+=$1} END {print sum}’ filename这个命令会计算文件中第一列的总和，并将结果打印出来。

  1. 将指定符号作为分隔符：

awk -F: ‘{print $1,$7}’ /etc/passwd这个命令会打印/etc/passwd文件中第一列和第七列的内容，使用“:”作为分隔符。
