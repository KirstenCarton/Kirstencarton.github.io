web259

考察的是php的soapclient，理解的是php里面如果要使用一个协议或者与服务器进项交互就需要使用soapclient这个工具，等价于python的request—

这篇文章写的很好，结合259讲解了soapclient的具体用法，搭配找到的另外几篇更加好食用

SoapClient原生类在开发以及安全中利用_合天网安实验室的博客-CSDN博客

一篇P神的关于crlf的文章：CRLF Injection漏洞的利用与实例分析 - phith0n (wooyun.js.org)

再总结一下读了一天的文章我的理解：

  • soapclient是个类似http的一种协议，需要在php.ini里面的extension=soapclient打开，可以向这个类里面传入wsdl（Web Services Description Language，也就是常用的xml，是个本地文件）参数，或者options参数
  • 为什么要在反序列化里面使用soapclient:当自定义的类里面没有可控的类的的时候，考虑使用php的原生类（也就是内置的已经写好的类）的相关魔术方法来达到目的
  • 为什么用soapclient：因为soapclient有个特性，它的__call方法会在调用不存在的方法的时候调用，这个是我们可控的
  • 这道题目怎么利用soapclient的__call方法：根据合天的那篇文章，当我们构造好options参数（就是一些http请求头uri,location,ua等等），使用soapclient进行发包的时候，我么可以看到一个http请求头SOAPAction这个是可控的，也就是我们可以在这请求头里面插入其他数据，于是就结合上了crlf的使用
  • 使用\r\n来拼接自己想要的http请求头，例如ua头等等，然后就开始构造
  • 现在已有的可控参数有：SOAPAction、User-Agent
  • 然后还需要考虑绕过content-type这个参数，因为这个限制了我们传入的是xml，想要使用application/x-www-form-urlencoded来传参数，所以考虑使用在content-type前面的UA拼\r\n来伪造content-type从而达到绕过的目的

于是有了payload：

  • 构造自己需要的ua头，然后调用一个soapclient类，其中为了应对题目中的explode和array_pop（这两个会要求截取xff头里面倒数第二个头作为自己的xff头，），所以传了两个127.0.0.1，第一个被pop出，第二个才会留下
  • 这里传入content-length让长度为13，于是token=ctfshow后面的数据就会被吃掉，让真正读取到的最后的东西是token=ctfshow
  • 注意payload里面需要将ua写成user_agent

传入后，然后访问flag.txt就行了

使用的是session来做反序列化的问题，就是默认三种格式php,php_serialize,php_binary,每种格式不一样，当编码的用的handler与解码用的handler不一样，就可以会造成漏洞的出现（能够注入）

php 键名+竖线+经过serialize()函数处理的值
php_binary 键名的长度对应的ascii字符+键名+serialize()函数序列化的值
