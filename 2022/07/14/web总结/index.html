1.信息泄露

思路：

  • view-source(这也是种协议)查看源代码
  • 访问index.phps
  • 查询robots.txt，检查是否有不允许爬取的文件目录
  • 访问有无.zip，.tz等后缀的文件（压缩文件）
  • 目录遍历
  • 备份文件下载
  • phpinfo 一般也可以在命令执行中使用phpinfo();函数来查看是否能使用shell

备份文件下载

当开发人员在线上环境中对源代码进行了备份操作，并且将备份文件放在了 web 目录下，就会引起网站源码泄露。

一般先使用扫描工具进行扫描，有nmap等工具

常见网站源码备份文件后缀

  • tar
  • tar.gz
  • zip
  • rar

常见网站源码备份文件名

  • web
  • website
  • backup
  • back
  • www
  • wwwroot
  • temp

5.ssrf

SSRF(Server-Side Request Forgery:服务器端请求伪造) 是一种由攻击者构造形成由服务端发起请求的一个安全漏洞。一般情况下，SSRF攻击的目标是从外网无法访问的内部系统。
